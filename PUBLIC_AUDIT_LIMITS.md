# Public Audit Limits

## Purpose

This repo is a staged public-facing window into the Indus lane. A clone can
establish only what the visible docs, starter package, and copied authority
surface support.

## What A Public Audit Can Establish

| Surface | What It Can Show |
| --- | --- |
| Source tree | the current staged repo shape, package root, and boundary docs |
| Copied authority docs | the carried-forward Phase 4, Phase 5, and paper verdict posture |
| Tagged or pinned snapshot | whether a given staged snapshot matches its own docs |
| Docs coherence | whether the front door, authority snapshot, and boundary docs agree |

## What A Public Audit Cannot Establish

| Missing Input | Why It Matters |
| --- | --- |
| full sign-image rights review | blocks unrestricted public image release claims |
| full monorepo rerun environment | blocks claiming complete replay from this starter alone |
| unpublished release metadata or owner-held contacts | blocks final public-release semantics |

## Rules For Interpreting This Repo

- Do not infer decipherment from morphology or language-like statistics.
- Do not turn search-demo usefulness into a separate product boundary.
- Do not treat later k=100 paper improvements as erasing the earlier Phase 4
  stability caveat.
- Treat `UNKNOWN`, `UNVERIFIED`, `PARTIAL`, and any explicitly owner-held
  release metadata as real states.

## Disputes

If a public statement appears to outrun the evidence surface, open an evidence
dispute and cite the exact path plus the missing or contradictory artifact.
