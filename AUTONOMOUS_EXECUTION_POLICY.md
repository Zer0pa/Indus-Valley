# Autonomous Execution Policy

This policy is the runtime operating contract for agents working inside
`gnosis-indus`.

## Governing Laws

1. The top acceptance gate for the active PRD is sovereign.
2. Runtime truth and artifact truth outrank prose.
3. Contradictions stay visible until resolved.
4. Runbook or phase plan before production code.
5. Falsify before promote.
6. Baseline loss is failure even if side metrics improve.
7. Keep the non-decipherment posture explicit.
8. Keep the Phase 4 stability caveat explicit.
9. Keep search framed as an in-repo application surface.
10. No toy demos, no proxy passes, no narrativized wins.

## Execution Mode

- Work end to end inside the assigned repo scope without interim reporting.
- Repo-local manifests, plans, summaries, logs, and reports are mandatory.
- User-facing reporting is end-only unless there is a blocker that cannot be
  removed locally or on admitted surfaces.
- Do not revert or overwrite unrelated work from other agents.

## Required Session Start Sequence

1. Read the PRD.
2. Read `AUTHORITY_SNAPSHOT.md`.
3. Read the local GPD pack.
4. Read the active phase files and machine state.
5. Confirm lane boundary, environment, current phase, and open blockers.

## What Counts As A Blocker

A blocker is real only if a required source, credential, dataset, hardware
surface, rights decision, or ownership boundary prevents the next step.

Ugly results, cautionary evidence, or a claim shrinking to match truth are not
blockers.

## Anti-Pattern Guard

Raise a blocker if any of these happen:

- coding before the plan exists
- promoting a claim without an evidence path
- treating search as a separate sovereign repo surface
- quietly dropping the Phase 4 stability caveat
- sliding into decipherment language
