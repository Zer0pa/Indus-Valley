# Support

## Routing

| Need | Route |
| --- | --- |
| Reproducible bug | GitHub bug report |
| Evidence or claim dispute | GitHub evidence dispute |
| Feature request | GitHub feature request |
| General question | GitHub question template |
| Security issue | private route in `../SECURITY.md` |

## What To Include

- repo version, branch, or commit
- exact path or artifact involved
- expected behavior versus observed behavior
- whether the issue depends on an external fetch surface or rights boundary

## Response Expectations

| Request Type | Expected Handling |
| --- | --- |
| Bug | best-effort triage while the repo remains staged |
| Question | best-effort response when the needed truth is local |
| Evidence dispute | compare the public statement against the copied authority docs |
| Feature request | review against roadmap and boundary docs |

## Boundary Note

Support replies do not change repo truth by themselves. Public claims change
only when the evidence surface and docs are updated together.
