# Indus Export Contract

## Purpose

Define the minimum truthful bundle boundaries for data, evidence, and
application surfaces exported from `gnosis-indus`.

## Export families

| Bundle | Required contents | Non-negotiable notes |
| --- | --- | --- |
| `catalogue_bundle` | catalogue summary, governing verdict, route-selection summary, stability summary, replay summary | must preserve the Phase 4 stability caveat |
| `falsification_bundle` | Phase 5 governing verdict plus comparison summaries | must preserve non-decipherment and substrate indistinguishability |
| `search_application_bundle` | search demo summary plus any future search interface docs | belongs inside `gnosis-indus`, not as a separate sovereign repo |
| `paper_bundle` | paper verdicts and later admitted source files | paper posture cannot overrule the repo's public caveats |

## Minimum semantic rules

1. Any exported morphology surface must say whether it is citing the k=70
   conditional review-pack catalogue, the later k=100 paper surface, or both.
2. Any exported language surface must say that linguistic structure does not
   equal decipherment or substrate closure.
3. Search exports must be described as application surfaces over admitted Indus
   bundles, not as generic search infrastructure.
4. Image-bearing exports require a rights review before public release.

## Consumer expectations

Downstream consumers should expect:

- explicit caveats, not washed-out marketing language
- local evidence or fetch references for every promoted file
- versioned bundle descriptions when the evidence basis changes
