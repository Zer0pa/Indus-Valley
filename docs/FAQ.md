# FAQ

## What is this repo?

It is the staged standalone repo candidate for the Indus lane inside the Gnosis
program: an anchor application and evidence surface, not a decipherment repo.

## How does this relate to the wider Gnosis portfolio?

This repo is the corpus-facing case study and application surface. It depends on
methods and evaluation work that may later live in sibling repos, but it should
not be described as a platform.

## What is actually verified here?

The copied authority bundle verifies the current scientific posture: a
stability-qualified Phase 4 catalogue, a non-decipherment Phase 5 verdict, a
useful in-repo search application surface, and the current paper verdict stack.

## What is still unknown or deferred?

The final public license, final contact surface, public image-bearing release
rights, and the first clean extracted runtime slice are still unresolved.

## Why does public documentation mention audit limits?

Because this repo is staged and rights-bounded. A reviewer can verify what the
docs and copied authority files show, but not everything the original monorepo
or unpublished assets could support.

## Where should I start if I want to inspect technical structure?

Read `docs/ARCHITECTURE.md`, then `code/README.md`, then the copied authority
docs named in `AUTHORITY_SNAPSHOT.md`.

## Where do I report defects, disputes, or questions?

Use the issue templates for bugs, evidence disputes, feature requests, and
questions. Use `SECURITY.md` for private vulnerability reporting.
