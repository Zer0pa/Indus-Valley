# Phase 01 Research

## Objective

Freeze the first real admission gate for `gnosis-indus`: source families, data
classes, search boundary, and the first extraction target.

## Carry-Forward Truth

- The repo starter is complete enough to govern work.
- The repo still lacks a real extracted runtime slice.
- Data-rights and heavy-asset limits are real.

## What This Phase Must Answer

1. Which code and artifact families are admitted first?
2. Which assets stay fetch-only because of rights or size?
3. What stronger smoke or replay path will Phase 02 need?
4. Which sibling-workstream boundaries must stay external?

## Decision Standard

Phase 01 passes only if a future extraction can start without ambiguity about
source ownership, data posture, or repo truth.
