# Conventions

## Purpose

This repo uses explicit conventions so the authority bundle, the repo boundary,
and the carried-forward scientific caveats cannot drift silently across phases
or agents.

## Core Project Conventions

- **Workspace root:** the directory that owns this `.gpd/` folder
- **Authority custody:** repo-local docs and copied authority files are
  sovereign unless the PRD admits a newer source
- **Authority metric:** `truth-surface coherence` measured by auditing README,
  authority, PRD, and boundary docs against the copied verdict surfaces
- **Baseline comparator:** copied authority docs plus corrected control-plane
  documents
- **Source ledger:** every promoted claim names a local source path or an
  explicit fetch surface
- **Benchmark discipline:** no core claim closes on proxy-only evidence
- **Reporting doctrine:** repo-local evidence is mandatory; outward reporting is
  blockers-only until the active gate is closed
- **Scope discipline:** search stays inside this repo; decipherment stays out

## Measurement Conventions

- **Authority threshold:** zero contradictory promoted claims; Phase 4 caveat,
  non-decipherment, and search boundary explicit in the front door
- **Comparator freeze point:** Phase 01
- **Replay policy:** starter smoke tests pass before Phase 02
- **Stress or falsification minimum:** no unsupported claim survives the docs
  and authority pass

## Naming Conventions

- **Workstream:** the standalone `gnosis-indus` repo candidate
- **Authority bundle:** copied review-pack and paper verdicts under `authority/`
- **Governing gate:** the highest non-negotiable repo extraction test in scope
- **Baseline:** the carried-forward verdict and control-plane truth
- **Artifact chain:** the path from copied source or extracted runtime to the
  promoted statement
